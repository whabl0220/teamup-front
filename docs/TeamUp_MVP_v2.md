# TeamUp MVP 문서 (v2)

## 1) 프로젝트 개요

- 서비스명: TeamUp
- 목표: 단톡방 조율 없이 원클릭으로 농구 매치 참여/인원 확인이 가능한 서비스
- 레퍼런스: 플랩풋볼의 소셜 매치 경험을 농구 도메인에 맞게 단순화

## 2) MVP 목표

- 사용자가 오늘/이번 주에 뛸 수 있는 매치를 빠르게 찾는다.
- 참가 신청/취소를 간단히 처리한다.
- 주최자가 인원 미달 매치를 운영 정책에 따라 취소하고, 참가자에게 알림을 보낸다.

## 3) 핵심 사용자

- 참가자: 빠르게 매치를 찾아 신청하고 싶은 일반 사용자
- 주최자(관리자): 초기 MVP 단계에서는 서비스 퀄리티 컨트롤을 위해 개발팀 내부 인원으로 권한을 제한하여 직접 매치를 생성하고 현장을 운영한다.

## 4) MVP 범위 (In Scope)

> 아래 항목은 현재 TeamUp 프론트 구현 기준으로 동기화함 (로컬 저장 기반 시뮬레이션 포함)

### 4.1 매치 리스트 및 상세 조회

- 날짜/시간 기준으로 매치 목록 조회
- 매치 카드 필수 정보: 매치 일시, 장소, 참가비, 현재 신청 인원/정원 (예: 12/15), 매치 레벨
- 장소 데이터: 전국 지도 API 연동 대신, 미리 정의된 로컬 구장(예: 세종대 대양AI센터 코트, 광진구 인근 공원 등 5~10곳) 목록에서 선택하는 하드코딩 방식으로 구현
- 매치 상세에서 시설/유의사항/취소 정책 확인

### 4.2 참가 신청 및 결제 확인 (수동화)

- 로그인 사용자가 매치에 참가 신청 가능
- 결제 방식: PG사 연동 없이, 주최자의 계좌(토스/카카오뱅크 등) 정보 안내 후 무통장 입금 방식으로 진행
- 상태값 흐름: 모집중 -> 유저 신청(입금 대기) -> 주최자 확인 후 승인(참가 확정) -> 정원 충족 시 마감
- 참가 취소 발생 시 빈자리 자동 반영 및 재모집 가능 상태 전환
- 상태값 종류: 모집중, 마감, 취소, 종료
- 운영자 액션 정책(현재 반영):
  - 신청 확정: `입금 대기(PENDING_DEPOSIT)` 상태에서만 가능
  - 환불 처리: `참가 확정(CONFIRMED)` 또는 `신청 취소(CANCELLED)` 상태에서만 가능
  - `마감(FULL)` 전환: 확정 인원 >= 정원일 때만 허용
  - `종료(ENDED)`된 매치는 다시 `모집중(RECRUITING)`으로 복귀 불가

### 4.3 프로필 (최소 정보)

- 닉네임
- 주력 포지션(가드/포워드/센터)
- 선호 플레이 스타일(자유 텍스트 또는 선택형)

### 4.4 주최자 인원 미달 취소 정책

- 인원 미달 시 주최자가 매치 취소 가능
- 100% 환불 가능 기한 내 취소를 원칙으로 운영 (계좌 이체를 통한 수동 환불 진행)
- 취소 처리 시 참가자에게 알림 발송

### 4.5 알림 (최소 이벤트 및 무료 매체 활용)

- 알림 발생 상황: 참가 신청 완료, 매치 취소(주최자/운영), 환불 처리 완료
- 알림 매체(현재 반영): 앱 내 토스트 + 로컬 알림 로그(`/notifications`)
- 로컬 알림 로그 저장소: `localStorage(teamup_notifications_v1)`
- 이메일(Clerk), 디스코드/슬랙 웹훅은 MVP 후속 연동 항목으로 보류

## 5) MVP 제외 범위 (Out of Scope)

- 자동 팀 밸런싱
- 레벨 자동 산정/매너 점수 고도화
- 대회/리그 기능
- 구장 예약(대관) 시스템 연동
- 팀 매치(팀 단위 대전) 고도화
- PG사 연동(신용카드 결제) 및 정교한 자동 정산 시스템
- 지도 API 기반 전국 구장 검색
- (보류) 이메일/외부 웹훅 실시간 알림 자동화

## 6) 주최자 혜택 가이드 (Phase 2 보류)

초기에는 개발팀이 직접 주최하므로 MVP에서는 제외한다.

추후 일반 유저에게 호스트 권한 개방 시, 수수료 할인/쿠폰, 무취소율 기반 배지 제공 등을 도입한다.

## 7) 사용자 핵심 플로우

### Flow A: 참가자

1. 홈에서 날짜별 매치 탐색
2. 상세에서 시간/장소/정원 및 입금 계좌 정보 확인
3. 참가 신청 클릭 (상태: 입금 대기)
4. 참가비 송금 후 대기
5. 주최자 승인 후 참가 확정 상태 확인 (현재는 앱 내 토스트/알림 로그 기준)
6. 필요 시 참가 취소 (기한 내 수동 환불 접수)

### Flow B: 주최자 (개발팀)

1. 정해진 프리셋 구장 중 하나를 선택해 매치 등록 및 모집 시작
2. 신청 현황 모니터링 (`/host/matches/[id]` 신청자 목록 및 상태 배지)
3. 참가자의 입금 내역 확인 후 수동으로 참가 확정 상태 변경
4. 정원 충족 시 마감 처리
5. 인원 미달 시 정책 시간 내 취소, 참가자 안내 및 수동 환불 진행

## 10) 현재 구현 상태 체크리스트 (2026-03-25)

- 완료: 홈/매치 목록/매치 상세/마이페이지 중심의 MVP v2 화면 정리
- 완료: 주최자용 매치 생성 화면 (`/host/matches/create`)
- 완료: 주최자용 신청자 관리 화면 (`/host/matches/[id]`) 및 상태 전이 정책 반영
- 완료: 참가자 상세 화면에서 내 신청 상태 및 매치 상태 배지 정리
- 완료: 이벤트 3종(신청 완료/매치 취소/환불 완료) 토스트 + 로컬 알림 로그 구현
- 완료: 백엔드 미연동 환경에서 동작하도록 매치/신청 로컬 저장 fallback 구성
- 보류: Clerk 이메일 알림 연동
- 보류: 디스코드/슬랙 웹훅 연동

## 8) 성공 지표 (MVP)

- 매치 상세 -> 참가 신청 전환율
- 신청 후 실제 참여율 (노쇼 비율)
- 인원 미달 취소율
- 취소 시 알림 전달 성공률

## 9) 문서 관리 규칙

- 향후 기획/정책/플로우 문서는 `docs` 폴더 아래에서 관리
- 본 문서는 MVP 기준 변경 시 버전 이력과 함께 업데이트
